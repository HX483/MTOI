# 奶茶店订单制作与原料库存管理系统用户手册

## 1. 系统概述

### 1.1 系统简介
奶茶店订单制作与原料库存管理系统是专为奶茶店设计的一体化管理平台，实现订单处理、原料库存、销售统计等核心业务的数字化管理，提高运营效率和服务质量。

### 1.2 系统功能
- **订单管理**：订单创建、处理流程、查询统计、异常处理
- **原料库存**：原料管理、库存监控、出入库操作、库存盘点
- **系统管理**：用户管理、角色权限、菜单管理、字典管理
- **报表分析**：销售报表、原料消耗报表、成本分析报表

### 1.3 系统特点
- 前后端分离架构，界面友好，操作便捷
- 完善的权限管理机制，确保数据安全
- 实时库存监控，智能预警提醒
- 丰富的报表分析，支持数据决策
- 响应式设计，支持多终端访问

## 2. 系统安装与配置

### 2.1 环境要求

#### 服务器环境
- **操作系统**：Windows Server 2016/2019 或 Linux（CentOS 7+/Ubuntu 18.04+）
- **CPU**：至少 2 核
- **内存**：至少 4GB
- **存储空间**：至少 50GB 可用空间

#### 软件环境
- **数据库**：MySQL 5.7+
- **缓存**：Redis 5.0+
- **Java**：JDK 1.8+
- **Node.js**：v18.0+

### 2.2 安装步骤

#### 2.2.1 数据库安装与配置
1. 安装MySQL 5.7 数据库
2. 创建数据库：`CREATE DATABASE mtoi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
3. 导入初始化数据：执行 `sql/mtoi.sql` 文件
4. 创建数据库用户并授权

#### 2.2.2 后端服务安装
1. 解压后端发布包到服务器目录
2. 修改 `application.yml` 配置文件，设置数据库连接信息
3. 启动后端服务：
   - Windows：运行 `start.bat`
   - Linux：执行 `./start.sh`
4. 验证服务：访问 `http://服务器IP:8080/doc.html`，查看 Swagger API 文档

#### 2.2.3 前端服务安装
1. 解压前端发布包到服务器目录
2. 配置 Nginx，指向前端静态文件目录
3. 启动 Nginx 服务
4. 验证服务：访问 `http://服务器IP`，显示系统登录界面

### 2.3 Docker 部署（推荐）

#### 2.3.1 环境准备
- 安装 Docker 和 Docker Compose

#### 2.3.2 部署步骤
1. 克隆项目代码到本地
2. 进入项目根目录
3. 执行部署命令：`docker-compose up -d`
4. 验证服务：
   - 前端：`http://localhost`
   - 后端：`http://localhost:8080`
   - 数据库：`localhost:3307`

## 4. 系统管理模块

#### 4.1.1 用户列表
- 功能：查看系统所有用户信息
- 操作：
  1. 进入「系统管理」→「用户管理」

  1. 点击「新增」按钮

## 5. 订单管理模块

### 5.1 订单创建

#### 5.1.1 堂食订单
- 操作：
  1. 进入「订单管理」→「创建订单」
  2. 选择订单类型：「堂食」
  3. 选择产品：
     - 在产品列表中选择需要的产品
     - 点击「+」按钮增加数量
     - 点击「-」按钮减少数量
     - 点击「编辑」按钮添加定制要求（如少糖、去冰）
  4. 确认订单信息
  5. 点击「创建订单」按钮完成订单创建

#### 5.1.2 外卖订单
- 操作：
  1. 进入「订单管理」→「创建订单」
  2. 选择订单类型：「外卖」
  3. 填写顾客信息：姓名、电话、配送地址
  4. 选择产品并添加定制要求
  5. 确认订单信息
  6. 点击「创建订单」按钮完成订单创建

#### 5.1.3 自提订单
- 操作：
  1. 进入「订单管理」→「创建订单」
  2. 选择订单类型：「自提」
  3. 填写顾客信息：姓名、电话
  4. 选择产品并添加定制要求
  5. 确认订单信息
  6. 点击「创建订单」按钮完成订单创建

### 5.2 订单处理流程

#### 5.2.1 待制作订单
- 功能：查看所有待制作的订单
- 操作：
  1. 进入「订单管理」→「待制作订单」
  2. 可以查看订单详情，包括产品、金额、定制要求等

#### 5.2.2 开始制作
- 操作：
  1. 在待制作订单列表中选择订单
  2. 点击「开始制作」按钮
  3. 订单状态变为「制作中」

#### 5.2.3 完成制作
- 操作：
  1. 在制作中订单列表中选择完成的订单
  2. 点击「完成制作」按钮
  3. 订单状态变为「已完成」

#### 5.2.4 订单配送
- 操作：
  1. 在已完成订单列表中选择需要配送的订单
  2. 点击「配送」按钮
  3. 订单状态变为「配送中」

#### 5.2.5 订单完成
- 操作：
  1. 在配送中订单列表中选择已送达的订单
  2. 点击「完成」按钮
  3. 订单状态变为「已完成」

### 5.3 订单查询与统计

#### 5.3.1 订单列表
- 功能：查询所有订单信息
- 操作：
  1. 进入「订单管理」→「订单列表」
  2. 设置查询条件：
     - 订单编号
     - 订单类型
     - 订单状态
     - 时间范围
  3. 点击「查询」按钮
  4. 可以导出订单数据为 Excel 文件

#### 5.3.2 订单详情
- 操作：
  1. 在订单列表中选择需要查看的订单
  2. 点击「详情」按钮
  3. 查看订单详细信息，包括产品、金额、状态历史等

#### 5.3.3 订单统计
- 功能：统计分析订单销售数据
- 操作：
  1. 进入「订单管理」→「订单统计」
  2. 设置统计条件：时间范围、统计维度
  3. 查看统计图表和数据
  4. 可以导出统计报表

### 5.4 异常订单处理

#### 5.4.1 取消订单
- 操作：
  1. 在订单列表中选择需要取消的订单
  2. 点击「取消订单」按钮
  3. 填写取消原因
  4. 点击「确认」按钮完成取消

### 6.2 库存管理

#### 6.2.1 库存列表
- 功能：查看所有原料的库存信息
- 操作：
  1. 进入「库存管理」→「库存列表」
  2. 可以通过原料名称查询库存
  3. 查看库存数量、预警阈值、更新时间等信息
  4. 低库存原料会显示红色预警标识

#### 6.2.2 原料入库
- 操作：
  1. 进入「库存管理」→「原料入库」
  2. 选择原料
  3. 填写入库信息：
     - 入库数量
     - 入库原因
     - 供应商
     - 备注
  4. 点击「保存」按钮完成入库
  5. 库存数量相应增加

#### 6.2.3 原料出库
- 操作：
  1. 进入「库存管理」→「原料出库」
  2. 选择原料
  3. 填写出库信息：
     - 出库数量
     - 出库原因
     - 备注
  4. 点击「保存」按钮完成出库
  5. 库存数量相应减少

#### 6.2.4 设置库存预警
- 操作：
  1. 在库存列表中选择需要设置预警的原料
  2. 点击「预警设置」按钮
  3. 输入预警阈值
  4. 点击「保存」按钮完成设置

### 6.3 库存盘点

#### 6.3.1 创建盘点单
- 操作：
  1. 进入「库存管理」→「库存盘点」
  2. 点击「创建盘点单」按钮
  3. 选择盘点范围：全部原料或部分原料
  4. 点击「生成盘点单」按钮

#### 6.3.2 执行盘点
- 操作：
  1. 在盘点单列表中选择需要执行的盘点单
  2. 点击「执行盘点」按钮
  3. 填写实际盘点数量
  4. 点击「提交」按钮完成盘点

#### 6.3.3 盘点差异处理
- 操作：
  1. 查看盘点差异报告
  2. 确认差异原因
  3. 点击「调整库存」按钮
  4. 库存数量根据盘点结果调整

## 9. 常见问题与故障排除

### 9.1 登录问题

#### 9.1.1 忘记密码
- 解决方法：联系系统管理员重置密码

#### 9.1.2 登录失败
- 可能原因：
  - 用户名或密码错误
  - 用户账号被禁用
  - 系统服务异常
- 解决方法：
  - 检查用户名和密码是否正确
  - 联系系统管理员确认账号状态
  - 检查系统服务是否正常运行

### 9.2 操作问题

#### 9.2.1 无法保存数据
- 可能原因：
  - 填写信息不完整或格式错误
  - 没有操作权限
  - 系统网络异常
- 解决方法：
  - 检查填写信息是否符合要求
  - 联系系统管理员确认权限
  - 检查网络连接

#### 9.2.2 页面加载缓慢
- 可能原因：
  - 网络带宽不足
  - 系统资源占用过高
  - 浏览器缓存问题
- 解决方法：
  - 检查网络连接
  - 关闭不必要的程序
  - 清除浏览器缓存

### 9.3 数据问题

#### 9.3.1 数据丢失
- 可能原因：
  - 系统故障
  - 操作错误
  - 数据库异常
- 解决方法：
  - 使用系统备份恢复数据
  - 联系系统管理员处理

#### 9.3.2 数据不一致
- 可能原因：
  - 并发操作冲突
  - 系统同步延迟
- 解决方法：
  - 刷新页面重试
  - 联系系统管理员处理
