# 奶茶店订单制作与原料库存管理系统需求说明书

## 1. 项目概述

本项目是一个基于MTOI框架结合大模型技术开发的奶茶店订单制作与原料库存管理系统，旨在帮助奶茶店实现订单处理、原料管理、数据分析等核心业务的数字化和智能化，提高运营效率和服务质量。

## 2. 功能需求

### 2.1 订单管理模块
- **订单创建与管理**：支持前台点单、外卖平台对接，订单状态实时更新
- **订单制作流程**：订单接收、制作、完成、配送全流程管理
- **订单历史查询**：支持多条件筛选、导出功能
- **订单统计分析**：销售数据、热销产品、时段分析等
- **异常订单处理**：退单、改单、投诉处理机制

### 2.2 原料库存管理模块
- **原料信息管理**：原料分类、规格、保质期等基础信息维护
- **库存实时监控**：当前库存数量、预警阈值设置
- **入库出库管理**：采购入库、生产领用、报损等操作记录
- **库存盘点功能**：定期盘点、差异调整
- **智能补货建议**：基于历史销售和库存消耗的补货提醒

### 2.3 大模型智能应用模块
- **智能推荐**：基于用户历史订单推荐个性化饮品
- **智能客服**：处理常见订单咨询和问题
- **配方智能优化**：基于原料成本和用户反馈优化配方
- **需求预测**：预测未来销售趋势和原料需求

### 2.4 系统管理模块
- **用户权限管理**：多角色权限控制（管理员、操作员、财务等）
- **数据字典维护**：饮品分类、原料分类等基础数据维护
- **系统参数配置**：工作时间、预警阈值等配置项
- **日志管理**：操作日志、系统日志记录与查询
- **数据备份恢复**：定期自动备份和手动恢复功能

### 2.5 报表分析模块
- **销售报表**：日报、周报、月报、年报
- **原料消耗报表**：各原料消耗趋势分析
- **成本分析报表**：原料成本、人工成本、利润分析
- **异常报表**：库存异常、订单异常等告警统计

## 3. 非功能需求

### 3.1 性能需求
- 系统响应时间：页面加载时间≤2秒，操作响应时间≤1秒
- 并发处理能力：支持50人同时在线操作，订单处理峰值≥100单/分钟
- 数据存储：支持至少3年的订单和库存数据存储

### 3.2 安全需求
- 数据加密：敏感数据传输和存储加密
- 访问控制：基于角色的细粒度权限控制
- 日志审计：关键操作记录完整审计日志
- 防SQL注入、XSS攻击等安全防护

### 3.3 可靠性需求
- 系统可用性≥99.9%
- 数据备份：每日自动备份，支持7天回滚
- 故障恢复：关键组件冗余，故障自动切换

### 3.4 易用性需求
- 界面简洁直观，符合操作习惯
- 支持快捷键操作
- 提供操作引导和帮助文档
- 响应式设计，支持不同终端访问

## 4. 技术需求

### 4.1 后端技术
- Java 1.8+
- Spring Boot 2.5.x
- MyBatis
- MySQL 5.7
- Redis缓存
- JWT身份认证

### 4.2 前端技术
- Vue 2.6.x
- Element UI
- Axios
- Vuex
- Vue Router

### 4.3 大模型集成(暂不开发)
- 支持DeepSeek等主流大模型API集成
- 实现智能推荐、客服、预测等功能接口

## 5. 数据需求

### 5.1 核心数据实体
- 用户（员工）信息
- 顾客信息
- 饮品信息（名称、配方、价格等）
- 原料信息（名称、规格、单位等）
- 订单信息（订单号、内容、状态、金额等）
- 库存信息（原料、数量、位置等）
- 采购记录
- 消耗记录

### 5.2 数据完整性约束
- 主键唯一性约束
- 外键关联约束
- 非空约束
- 业务规则约束（如库存不能为负数）

## 6. 验收标准

1. 所有功能按照需求规格实现，无遗漏
2. 系统运行稳定，无严重bug
3. 性能满足需求指标
4. 安全测试通过
5. 用户培训完成，能够正常使用系统

## 7. 风险评估

1. **技术风险**：大模型集成复杂度高，可能需要多次调试优化
2. **业务风险**：实际业务流程可能与需求分析存在差异
3. **数据安全风险**：涉及订单和支付信息，需加强安全防护
4. **系统兼容性风险**：不同设备和浏览器的兼容性问题

## 8. 项目范围限定

1. 本系统不包含支付功能，仅处理订单信息
2. 外卖平台对接仅提供数据接口，具体对接由第三方完成
3. 大模型功能作为辅助，不替代人工决策
4. 系统支持PC端访问，不开发原生APP
